# GeoSlope 开发工作流

## 1. 问题定义

开发 2D 边坡稳定分析工具，用 FEM 强度折减法计算安全系数，并用 LEM 极限平衡法交叉验证。

- **输入**：边坡几何（高/角度）、材料参数（c/φ/γ/E/ν）
- **输出**：安全系数 FoS、位移云图、塑性区、滑弧
- **验证**：FEM 与 LEM 结果应一致（偏差 < 10%）

## 3. 调试：FEM 高估 FoS

### 现象

> FEM FoS = 2.144，LEM 共识 = 1.197，偏差 +79%

### 逐步排查

**Step 1 — 确认 LEM 内部一致性**

4 种方法结果：Fellenius 1.136 / Bishop 1.219 / Janbu 1.253 / M-P 1.180
标准差仅 0.070（5.8%），LEM 内部高度一致 → 问题不在 LEM。

**Step 2 — 检查 FEM 塑性区分布**

弹性解（FoS=1.0）时 753/866 单元被判为塑性 → 不可能。H=10m、c=15kPa 的 45° 边坡在弹性状态下大部分区域不应屈服。

**Step 3 — 定位屈服函数计算**

Mohr-Coulomb 屈服函数 f = (σ1 - σ3) - (σ1 + σ3) * sinφ - 2c*cosφ

关键：σ1 和 σ3 怎么算的？

**Step 4 — 发现根因**

`material.py` 中 `_principal_stresses()` 把出平面应力硬编码为 0：

```python
s3 = 0.0  # 错误：平面应变 ≠ 平面应力
```

平面应变条件下，出平面方向有约束 εz=0，因此：

```
σz = ν(σx + σy)  # 泊松效应
```

硬编码 s3=0 导致最大主应力被严重高估，屈服函数过大约 140 kPa。

### 修复

```python
def _principal_stresses_ps(sigma, nu):
    s_in_1, s_in_2 = ...  # 面内主应力
    s_out = nu * (sx_g + sy_g)  # 出平面应力（平面应变）
    all_s = [s_in_1, s_in_2, s_out]
    all_s.sort(reverse=True)
    return all_s[0], all_s[-1]  # σ1 (max), σ3 (min)
```

### 连带修复

修复主应力后，弹塑性求解器出现新问题：损伤折减过于激进（factor=0.01），单元刚度一次折减 99%，导致数值崩溃。

改进方案：
- 渐进损伤：`damage[e] = damage[e] * (1-rate) + target * rate`，rate=0.5
- 位移失效判据：`max_disp > 0.02 * slope_height` → 判定失稳
- 收敛条件：塑性区稳定 + 位移稳定

SRF 搜索改进：
- 增量扫描确定初始界：`fos = 0.5, 0.7, 0.9, ...` 找到收敛→不收敛边界
- 二分法求精：20 步二分至 1e-4 精度

### 结果

| 指标 | 修复前 | 修复后 |
|------|--------|--------|
| FEM FoS | 2.144 | 1.256 |
| LEM 共识 | 1.197 | 1.197 |
| 偏差 | +79% | +5.0% |
| 判定 | FAIL | **AGREE** |

## 5. 方法论总结

```
问题 → 假设 → 验证 → 定位 → 修复 → 验证
 ↑________________________________________|
```

- **FEM 高估 FoS** → 先确认 LEM 没问题（排除嫌疑）→ 检查中间状态（弹性解塑性区异常）→ 追到主应力计算 → 修复原理性错误
- **弹塑性不稳定** → 从物理直觉判断（一次折减 99% 不现实）→ 引入渐进机制
- **SRF 搜索边界** → 先粗扫后精二分 → 鲁棒

### 关键原则

1. **先排除，再定位**：LEM 一致 → 问题在 FEM；塑性区异常 → 问题在屈服判据
2. **中间状态诊断**：不只看最终结果，看弹性解/耦合步的中间状态
3. **物理直觉**：岩土力学中 s3=0 只适用于地表的平面应力，10m 边坡是平面应变
